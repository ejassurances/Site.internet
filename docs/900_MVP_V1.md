# 900 — MVP V1 EJ Assurances

**Version :** 1.0.0  
**Statut :** En vigueur  
**Propriétaire :** Direction EJ Assurances  

## Objectif

Créer une première version utilisable rapidement pour gérer les premiers prospects et clients EJ Assurances.

## Priorité V1

La V1 doit permettre de :

1. créer un prospect ;
2. suivre un prospect ;
3. transformer un prospect en client ;
4. créer un projet client ;
5. déposer des documents ;
6. suivre les tâches ;
7. historiser les échanges ;
8. préparer les documents conformité de base.

## Modules V1 obligatoires

### Site public

- Accueil
- Expertises
- Coparentalité
- Contact
- Prendre rendez-vous
- Mentions légales
- Politique de confidentialité

### CRM

- Prospects
- Clients
- Projets
- Tâches
- Documents
- Historique des échanges

### Conformité de base

- DER
- Recueil des besoins
- Devoir de conseil
- Preuve de remise des documents

### Espaces

- Admin courtier
- Client
- Mandataire
- Prescripteur

## Données sensibles

Ne jamais collecter l’orientation sexuelle ou toute donnée sensible inutile.

Le CRM doit uniquement collecter les données nécessaires à l’accompagnement, à la conformité et à la gestion du dossier.

## Hors périmètre V1

- IA avancée
- Orchestrateur IA
- API devis compagnies
- Dashboard avancé
- Automatisations complexes
- Scoring avancé
- Reporting financier

## Règle MVP

Toute fonctionnalité qui n’aide pas à gérer un vrai prospect ou un vrai client immédiatement est reportée.
