# Base de données

## Objectif du fichier
Décrire la structure de données et les principales entités nécessaires pour la plateforme.

## Décisions validées
- Centraliser les données clients, contrats, interactions et utilisateurs.
- Prévoir une table de contenus pour les pages publiques, le centre de connaissances et les actualités.
- Gérer les accès et les profils utilisateurs dans des entités dédiées.
- Prévoir des champs SEO et géolocalisation pour les pages SEO/GEO.

## Éléments à préciser plus tard
- Le modèle de données exact pour les audiences et les rôles.
- Le schéma des relations entre clients, contrats, leads et commissions.
- Les indexes et les stratégies de performances.
- Le stockage des médias et des ressources (images, documents, vidéos).
