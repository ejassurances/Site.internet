# Architecture globale

## Objectif du fichier
Donner une vue d’ensemble de l’architecture fonctionnelle et technique de la plateforme.

## Décisions validées
- Séparer clairement le site public, le centre de connaissances, les actualités et les espaces connectés.
- Prévoir une architecture modulaire pour faciliter les évolutions.
- Utiliser une base de données centralisée avec des espaces connectés distincts.
- Prévoir une logique de contenu SEO adaptée à chaque type de page.

## Éléments à préciser plus tard
- Le choix final de la stack technique pour le back-end et le front-end.
- Les briques techniques exactes (frameworks, CMS, services externes).
- L’organisation des API et des micro-services, le cas échéant.
- Les schémas d’infrastructure et de déploiement.
