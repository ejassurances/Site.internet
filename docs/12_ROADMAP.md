# 12 - Roadmap

## Objectif

Définir une trajectoire progressive de construction de la plateforme EJ Assurances.

La plateforme doit évoluer du site public vers un CRM métier complet, conforme et assisté par IA.

## Phase 1 - Documentation, architecture, identité, pages publiques, SEO/GEO

Objectifs :

- stabiliser le positionnement du cabinet ;
- structurer la documentation stratégique ;
- clarifier l'architecture globale ;
- moderniser l'identité visuelle ;
- produire les pages publiques prioritaires ;
- optimiser le contenu pour le SEO et les moteurs génératifs.

Livrables :

- documentation projet ;
- architecture cible ;
- site public ;
- pages d'expertise ;
- formulaire de préqualification ;
- premières ressources pédagogiques.

## Phase 2 - CRM de base, prospects, clients, documents, tâches, emails

Objectifs :

- créer le socle CRM métier ;
- centraliser les prospects et clients ;
- suivre les demandes entrantes ;
- rattacher documents et échanges ;
- préparer la gestion des emails.

Livrables :

- fiches prospects ;
- fiches clients ;
- documents ;
- tâches ;
- historique de relation ;
- premiers tableaux de bord.

## Phase 3 - Espaces connectés client, courtier, collaborateur, mandataire, prescripteur

Objectifs :

- créer les espaces par rôle ;
- limiter les accès selon les droits ;
- permettre le suivi des dossiers ;
- fluidifier la collaboration interne et externe.

Livrables :

- espace client ;
- espace courtier admin ;
- espace collaborateur ;
- espace mandataire ;
- espace prescripteur ;
- espace prospect.

## Phase 4 - Conformité ACPR/DDA, devoir de conseil, recueil des besoins, audit logs

Objectifs :

- intégrer les obligations réglementaires au CRM ;
- structurer le recueil des besoins ;
- produire et archiver le devoir de conseil ;
- conserver les preuves ;
- journaliser les actions sensibles.

Livrables :

- DER ;
- recueil des besoins ;
- module devoir de conseil ;
- recommandations personnalisées ;
- gestion des consentements ;
- audit logs ;
- classeur ACPR.

## Phase 5 - Agents IA Mistral assistifs, relation client, emails, sinistres, conformité

Objectifs :

- intégrer des agents IA encadrés ;
- assister la relation client ;
- aider au traitement des emails ;
- soutenir le suivi des sinistres ;
- préparer des brouillons de conformité.

Livrables :

- assistant commercial ;
- assistant relation client ;
- assistant email ;
- assistant sinistre ;
- assistant conformité ;
- gouvernance IA ;
- journalisation IA.

## Phase 6 - Automatisation avancée, diagnostics, tableaux de bord, pilotage cabinet

Objectifs :

- automatiser les relances non sensibles ;
- améliorer les diagnostics ;
- construire des tableaux de bord de pilotage ;
- suivre la performance commerciale, opérationnelle et conformité.

Livrables :

- workflows avancés ;
- alertes ;
- relances ;
- dashboards cabinet ;
- indicateurs de production ;
- indicateurs conformité ;
- pilotage mandataires et prescripteurs.

## Principes de priorisation

- conformité avant automatisation ;
- sécurité avant confort ;
- validation humaine avant IA ;
- données structurées avant reporting ;
- expérience client claire avant complexité interne.
