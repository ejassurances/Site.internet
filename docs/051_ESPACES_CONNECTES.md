PARTIE 1 — Philosophie des espaces connectés
1. Objet du document

Ce document décrit l'ensemble des espaces connectés de la plateforme EJ Assurances.

Il définit :

les différents espaces utilisateurs ;
leurs objectifs ;
leur organisation ;
leurs interactions ;
leurs droits d'accès.

Chaque utilisateur dispose d'un espace adapté à son rôle tout en travaillant sur les mêmes données.

2. Philosophie générale

La plateforme ne possède pas plusieurs applications.

Elle possède une seule plateforme composée de plusieurs espaces.

Tous les espaces utilisent :

les mêmes données ;
les mêmes Projets ;
les mêmes Documents ;
les mêmes Workflows ;
les mêmes Agents IA.

Seule la présentation évolue selon l'utilisateur.

3. Les espaces de la plateforme

La plateforme est organisée autour de plusieurs espaces principaux.

Site public

Présentation du cabinet.

Acquisition de prospects.

Prise de rendez-vous.

Création des premiers Projets.

Espace Prospect

Permet au Prospect de compléter son dossier avant de devenir Client.

Espace Client

Permet au Client de suivre l'ensemble de sa relation avec le cabinet.

Espace Mandataire (MIA)

Permet au Mandataire de développer et gérer son activité.

Espace Collaborateur

Permet aux équipes du cabinet de traiter les Projets.

Espace Administration

Permet de piloter le cabinet.

Espace Partenaire (optionnel)

Permet à certains partenaires d'échanger avec le cabinet.

Espace Supervision IA

Permet de piloter les Agents IA.

4. Une seule authentification

Chaque utilisateur possède un compte unique.

Après authentification, la plateforme détermine automatiquement :

son rôle ;
ses autorisations ;
son espace par défaut ;
ses tableaux de bord.

Un utilisateur possédant plusieurs rôles peut basculer d'un espace à un autre sans se reconnecter.

5. Une navigation cohérente

Tous les espaces utilisent les mêmes principes de navigation.

On retrouve notamment :

le menu principal ;
la recherche globale ;
les notifications ;
les raccourcis ;
le centre d'aide ;
le profil utilisateur.

L'objectif est qu'un utilisateur retrouve immédiatement ses repères.

6. Les éléments partagés

Tous les espaces peuvent accéder, selon les droits, aux mêmes objets métiers :

Client ;
Projet ;
Contrat ;
Produit ;
Partenaire ;
Document ;
Interaction ;
Tâche ;
Sinistre ;
Réclamation.

Les données ne sont jamais dupliquées.

7. Communication entre les espaces

Une action réalisée dans un espace est immédiatement visible dans les autres espaces autorisés.

Exemple :

Le Client dépose un document.

↓

Le Collaborateur le voit immédiatement.

↓

Le Mandataire reçoit une notification.

↓

Le Projet est mis à jour.

↓

Le Workflow continue.

↓

L'Agent IA contrôle automatiquement la complétude.

Tout repose sur la même donnée.

8. Objectif

Les espaces connectés poursuivent un objectif unique :

Permettre à chaque utilisateur de travailler dans une interface adaptée à sa mission, tout en partageant la même réalité métier.

9. Critères de validation

L'organisation des espaces sera considérée comme conforme lorsque :

tous les utilisateurs travaillent sur une plateforme unique ;
les données sont partagées sans duplication ;
chaque rôle dispose d'une interface adaptée ;
les interactions entre espaces sont automatiques ;
la navigation reste homogène ;
les droits d'accès sont respectés.

Sa mission n'est pas de vendre.

Sa mission est de faire avancer les Projets.

Le Collaborateur doit pouvoir identifier immédiatement :

les dossiers en attente ;
les actions à réaliser ;
les blocages ;
les priorités ;
les demandes des conseillers ;
les demandes des clients.

Son espace est orienté production, qualité et rapidité d'exécution.

PARTIE 5 — L'Espace Client (Le coffre-fort numérique de la relation)

L'Espace Client n'est pas un simple portail où télécharger des documents.

Il constitue l'espace personnel du Client tout au long de sa relation avec le cabinet.

Le Client ne doit jamais avoir besoin d'envoyer un email pour savoir où en est son dossier.

Il doit retrouver toutes les informations importantes au même endroit.

L'objectif est de renforcer la confiance, la transparence et l'autonomie du Client.

PARTIE 6 — L'Espace Prospect
53. Philosophie

L'Espace Prospect constitue la continuité naturelle du site internet.

Il permet au Prospect de commencer son parcours sans avoir encore la qualité de Client.

L'objectif est de supprimer les échanges inutiles par email et de guider le Prospect étape par étape.

Le Prospect doit ressentir qu'il est déjà accompagné, tout en comprenant qu'il est au début de son parcours.

54. Création automatique

L'espace Prospect est créé automatiquement lorsqu'un Prospect :

complète un formulaire du site ;
demande un devis ;
prend un rendez-vous ;
est créé par un Mandataire ;
est créé par un Collaborateur ;
est importé depuis une campagne.

Aucune création manuelle d'espace n'est nécessaire.

Les mêmes échanges sont conservés.

62. Critères de validation

le Prospect peut suivre son Projet en autonomie ;
les formulaires sont intégrés à la plateforme ;
Google Workspace fonctionne uniquement en arrière-plan ;
les documents sont déposés directement dans le Projet ;
la Lettre de Mission est générée automatiquement ;
le passage vers l'Espace Client est transparent ;
les Agents IA accompagnent le Prospect sans se substituer au conseiller.


PARTIE 7 — L'Espace Partenaire
63. Philosophie

L'Espace Partenaire est un espace optionnel destiné aux partenaires avec lesquels le cabinet souhaite collaborer de manière numérique.

Il ne remplace pas les extranets des compagnies ou des grossistes.

Il facilite les échanges entre EJ Assurances et ses partenaires lorsque cela est pertinent.

Cet espace permet de fluidifier les traitements, réduire les échanges d'emails et améliorer le suivi des dossiers.

64. Les partenaires concernés
