# 79 - Devoir de conseil

## Objectif

Définir le module de devoir de conseil.

Le module doit permettre de documenter la recommandation du cabinet, les besoins exprimés, les solutions étudiées et la preuve de remise au client.

## Informations à recueillir

- informations client ;
- besoins et exigences ;
- situation familiale ;
- situation professionnelle ;
- situation financière ;
- contrats existants ;
- objectifs ;
- risques identifiés ;
- contraintes ;
- préférences ;
- niveau de compréhension.

## Analyse

Le devoir de conseil doit présenter :

- solutions étudiées ;
- solution recommandée ;
- justification de la recommandation ;
- alternatives ;
- exclusions ou limites ;
- points de vigilance ;
- documents utilisés ;
- hypothèses retenues.

## Validation

Règles :

- validation humaine obligatoire ;
- validation par courtier habilité ;
- versionnage ;
- horodatage ;
- preuve de remise ;
- archivage.

## Rôle de l'IA

L'IA peut aider à :

- préparer un brouillon ;
- structurer les informations ;
- détecter les champs manquants ;
- résumer les risques ;
- proposer une formulation.

L'IA ne peut pas :

- valider le conseil ;
- décider de la recommandation ;
- remettre le document au client sans validation ;
- modifier un contrat seule.

## Archivage

Le CRM doit conserver :

- version du document ;
- auteur ;
- validateur ;
- date de validation ;
- date de remise ;
- preuve de remise ;
- acceptation ou refus client.

## Points ouverts

- modèle final ;
- niveau de signature ;
- format PDF ;
- durée de conservation ;
- contrôle qualité interne.
